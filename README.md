# Threshold ElGamal Voting Fe

Fe implementation of the on-chain contract layer for threshold ElGamal voting
over BN254.

The design follows the architecture described in Shutter's article
[Coming Soon to DAOs: Permanent Shielded Voting via Homomorphic Encryption](https://blog.shutter.network/coming-soon-to-daos-permanent-shielded-voting-via-homomorphic-encryption/):
votes are encrypted with ElGamal in the exponent, valid ciphertexts are added
homomorphically, and only the final aggregate tally is threshold-decrypted.

This repository is narrower than the full Shutter system. It focuses on Fe
contracts and BN254/EVM mechanics: storing the election public key, accepting
ciphertext ballots, maintaining the encrypted aggregate, and recording the final
result. It does not include Shutter Keyper infrastructure, DKG, Snapshot
integration, voter authentication, ballot-validity ZK proofs, or on-chain DLEQ
verification for decryption shares.

On `extra`, this repository also includes the Python oracle, deterministic
vectors, Fe vector tests, Foundry benchmarks, generated bytecode, and validation
reports used for handoff.

## Layout

- `ingots/crypto/` - reusable BN254 G1, ElGamal, fixed-array aggregation, and
  threshold-decryption helper functions.
- `ingots/voting/` - `ThresholdElection`, the Fe contract that stores election
  state and aggregates encrypted ballots.
- `ingots/vector_tests/` - deterministic vector and contract integration tests
  on the `extra` branch.
- `python_reference/` - executable reference model and test vector generator on
  the `extra` branch.
- `bench/` - Foundry gas benchmark harness on the `extra` branch.
- `README.md` and `fe.toml` - repo-root workspace entry point; commands run from
  the repository root.

## Build

```bash
fe build
```

## Extra branch

```bash
git checkout extra
fe test ingots/vector_tests
fe build
```

Use `master` for the compact implementation branch and `extra` when you want
the validation material that is useful for handoff and benchmarking.
