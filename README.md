# Threshold ElGamal Voting Fe

Fe implementation of the on-chain contract layer for threshold ElGamal voting over BN254.

The design follows the architecture described in Shutter's article [Coming Soon to DAOs: Permanent Shielded Voting via Homomorphic Encryption](https://blog.shutter.network/coming-soon-to-daos-permanent-shielded-voting-via-homomorphic-encryption/): votes are encrypted with ElGamal in the exponent, valid ciphertexts are added homomorphically, and only the final aggregate tally is threshold-decrypted.

This repository is narrower than the full Shutter system. It focuses on Fe contracts and BN254/EVM mechanics: storing the election public key, accepting ciphertext ballots, maintaining the encrypted aggregate, and recording the final result. It does not include Shutter Keyper infrastructure, DKG, Snapshot integration, voter authentication, ballot-validity ZK proofs, or on-chain DLEQ verification for decryption shares.

The reference material, Python oracle, deterministic vectors, Fe vector tests, benchmarks, generated bytecode, reports, and local validation artifacts live on the `extra` branch.

## Layout

- `ingots/crypto/` - reusable BN254 G1, ElGamal, fixed-array aggregation, and threshold-decryption helper functions.
- `ingots/voting/` - `ThresholdElection`, the Fe contract that stores election state and aggregates encrypted ballots.
- `README.md` and `fe.toml` - repo-root workspace entry point; commands run from the repository root.

## Build

Tested with `fe 26.2.0`.

```bash
fe build --emit bytecode,runtime-bytecode,abi,metadata
```

## Gas Snapshot

The gas benchmarks live on `extra` and compare Fe's Sonatina backend with Fe to Yul to solc. At a high level, `castVote(...)` is the expensive path because each vote validates two BN254 points and updates both ciphertext points in storage. Subsequent votes are much cheaper than the first vote, `getAggregate()` is relatively small, and the optimized Sonatina and Yul paths are close enough that backend choice is not the main driver here.

See the full gas report on `extra`: [reports/FOUNDRY_GAS_REPORT.md](https://github.com/g-r-a-n-t/threshold-elgamal-voting-fe/blob/extra/reports/FOUNDRY_GAS_REPORT.md).
